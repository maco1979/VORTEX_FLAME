# 9灵魂学习机制 - 系统化知识分配与训练路径

## 核心原则

1. **先基础后应用** - 每个灵魂必须先掌握基础学科，再学关联学科，最后学行业应用
2. **低→中→高递进** - 每个级别内部再分初级/中级/高级
3. **数据质量第一** - 禁止模板化数据、禁止随机数字、禁止空洞填充语
4. **物理定律验证** - 所有训练数据必须符合物理定律和数学逻辑

## 4层知识架构

```
Layer 1: 基础学科 (UNESCO 7大学科)
  ↓
Layer 2: 技术科学 (5大技术领域)
  ↓
Layer 3: 行业门类 (GB/T 20个门类)
  ↓
Layer 4: 具体行业 (473个中类)
```

## 训练数据质量标准

### 禁止出现的内容
- ❌ "典型错误包括：过度推广、忽略约束、混淆概念"
- ❌ "步骤1：分析问题→步骤2：验证结果→步骤3：推广应用"
- ❌ "假设初始值为XX，变化量为XX，计算最终结果"
- ❌ "爱因斯坦建议分步骤验证" 等空洞填充语
- ❌ "这种同构映射是爱因斯坦跨界思维的基础" 等模板总结
- ❌ 任何随机数字计算题
- ❌ 重复的模板化句式

### 必须包含的内容
- ✅ 真实的物理/数学公式和常数
- ✅ 具体的数值和单位
- ✅ 逻辑严密的推导过程
- ✅ 实际的实验或应用案例
- ✅ 每条数据独立完整，不依赖上下文

### 数据格式
```json
{
  "data": [
    {
      "text": "完整的知识点描述，包含公式、常数、推导",
      "level": "1a|1b|1c|2|3"
    }
  ]
}
```

## 课程学习顺序

```
Phase 1: 基础初级 (1a) - 5%数据量
  目标: Loss < 4.0
  内容: 基本概念、定义、公式

Phase 2: 基础中级 (1b) - 20%数据量
  目标: Loss < 2.5
  内容: 理论推导、公式应用

Phase 3: 基础高级 (1c) - 10%数据量
  目标: Loss < 2.0
  内容: 高级理论、前沿概念

Phase 4: 关联深化 (2) - 35%数据量
  目标: Loss < 1.8
  内容: 跨学科关联、交叉创新

Phase 5: 行业应用 (3) - 30%数据量
  目标: Loss < 1.5
  内容: 实际应用、行业案例
```
