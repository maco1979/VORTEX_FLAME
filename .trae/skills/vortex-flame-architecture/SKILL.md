## 第2步：蒸馏压缩9个主模型

```
目标: 9个大脑先定型、提纯、轻量化
方法: 知识蒸馏 + INT8量化
  - 知识蒸馏: 150L大模型 → 蒸馏到更紧凑的150L (去掉冗余，保留逻辑链路)
  - INT8量化: FP16 → INT8, 体积减半，精度损失<1%
  - 150L FP16=2040MB → 蒸馏+INT8 ≈ 500-700MB
流程:
  1. 用150L大模型做teacher，生成软标签
  2. 用软标签+硬标签训练紧凑版student
  3. AB测试: student vs teacher，确认无损
  4. INT8量化student
  5. 再次AB测试
原则:
  - 先蒸馏大的！大的先定型、先提纯
  - 大的定型后稳定不动，后面才不会乱套
  - 核心思考能力一点不掉
验收: 压缩后Loss变化 < 2%，推理速度提升 > 50%
```

**为什么先蒸馏大的**:
- 思维层先固化、提纯，逻辑永远纯净
- 大的定型后做小的，小的可以对着大的学（蒸馏）
- 大的先轻量化，后面训小的时显存更宽裕

## 第4步：蒸馏压缩技能小模块

```
目标: 全部小模型轻量化
方法: 知识蒸馏 + INT8量化
  - 小AI用蒸馏好的主脑做teacher
  - 12L蒸馏+INT8: ~30-50MB/个
  - 45个小AI × 40MB = 1.8GB 总计
流程:
  1. 小AI训练完成 → 用主脑做teacher蒸馏
  2. INT8量化
  3. AB测试每个小AI
原则:
  - 小模型独立蒸馏，互不影响
  - 坏一个换一个，不影响9个核心
  - 可单独升级、替换
验收: 压缩后各小AI准确率变化 < 3%
```

# 为什么"先蒸馏大的→再做小的→蒸馏小的→对接"

| 步骤 | 做什么 | 为什么 |
|------|--------|--------|
| 先蒸馏大 | 大模型先定型、提纯 | 逻辑纯净，不被技能干扰 |
| 再做小的 | 用蒸馏好的大模型当teacher | 小的对着大的学，质量更高 |
| 蒸馏小的 | 小模型也轻量化 | 体积小、速度快、好部署 |
| 对接 | 蒸馏好的大+小连接 | 各自独立，插拔自由 |

| 对比项 | 分步蒸馏(我们的方案) | 统一压缩(错误方案) |
|--------|---------------------|-------------------|
| 逻辑纯净度 | 大的先提纯，逻辑永远纯净 | 技能参数污染核心思维 |
| 风险 | 每部分可控、可测试 | 层间串线、权重混乱 |
| 可维护性 | 坏一个换一个 | 牵一发动全身 |
| Teacher质量 | 大的先蒸馏好，小的学得更好 | 没有好teacher，小的学不好 |
| 升级 | 单独升级任意模块 | 全部重训 |
