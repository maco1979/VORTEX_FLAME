================================================================================
  第四部分：自博弈系统(HERMES AlphaZero)
================================================================================

4阶段循环:
  Phase 0: 自检(GPU/LoRA/磁盘/参数)
  Phase 1: 自博弈出题(1000题) + 自答 + 自判(WIN/LOSE)
  Phase 2: DPO配对(WIN=chosen, LOSE=rejected)
  Phase 3: SFT Warmup(30%WIN) + DPO训练
  Phase 4: 评估 + 回归测试 + 三重门禁

关键参数:
  MAX_ITERATIONS = 8 (iter1-5自博弈 + iter6-8 Prompt策略自迭代)
  QUESTIONS_PER_ITER = 1000
  PASS_THRESHOLD_SCHEDULE = {1:5, 2:5, 3:6, 4:6, 5:7, 6:7, 7:8, 8:8} (v2: +1补偿vsa_bonus 0-3)
  SELFPLAY_MAX_ITER = 5, PROMPT_ITER_START = 6
  DPO_BETA = 0.1
  LORA_R = 16, LORA_ALPHA = 32
  LR = 2.5e-5, MAX_GRAD_NORM = 0.3
  GRADUATION_THRESHOLD = 0.95 (连续3轮>95%→毕业)

DPO训练原理:
  ref_model从SFT warmup后权重同步(4bit NF4)
  L_DPO = -log σ(β · (log π_θ(chosen|x)/π_ref(chosen|x)
                       - log π_θ(rejected|x)/π_ref(rejected|x)))
  V100-16GB显存: 4bit policy + 4bit ref ≈ 11.5GB

18项安全措施:
  梯度: max_grad_norm=0.3, 4bit NF4, fp16, cosine LR, warmup=10
  过拟合: lora_dropout=0.05, weight_decay=0.01, DPO beta=0.1, MAX_DPO_LOSS=3.0, MIN_DPO_LOSS=0.01
  欠拟合: LR衰减1.3^iter, DPO LR减半, MIN_DPO_PAIRS=5
  NaN: NaN检测+门禁(立即回滚)
  DPO配对: 最短20字符, 长度比<1.5, chosen≠rejected, ref_model冻结

元认知闭环(反思→策略→执行):
  反思: 分析最近200条LOSE轨迹，提取缺失关键词(top5)
  策略: pass_rate<30%→intensive_drill, <50%→targeted_practice,
        <70%→moderate, >95%→graduate, >90%→maintenance
  执行: 调整topic_weights(EMA old×0.6+new×0.4), 10%自生成出题, 毕业后权重0.1恢复

VSA语义评分(v2重构 2026-05-21):
  FHRR编码: VSA_DIM=2048, VSA_SEED=42, _stable_hash确定性哈希
  返回值: (combined, global_sim) — combined用于External评分, global_sim用于AvgVSA追踪
  combined = FHRR全局相似度×0.4 + 关键词覆盖率×0.6 (仅External维度使用)
  global_sim = 纯FHRR语义相似度 (所有维度用于AvgVSA统计和vsa_bonus)
  vsa_bonus = min(3, int(global_sim*15)) — 语义深度奖励0-3分
  AvgVSA = 所有任务global_sim均值 — 真正反映语义深度(不被关键词覆盖率污染)
  词边界匹配: _word_boundary_match替代子串匹配(CJK走in快速路径)
  Cleanup二次验证: 分数=threshold-1时触发VSA评分(global_sim>0.40翻转)
  Einstein翻转阈值: global_sim>0.35
  所有维度(含Algorithm/Debug)都计算VSA, 不再返回0.0
  trajectory存储reference字段, re-eval时正确计算VSA

代码执行验证:
  extract_python_code → sandbox_exec(10秒超时) → build_test_cases → 评分
  Algorithm/Debug/Complexity维度优先使用代码执行验证
  HARD_CONSTRAINTS: Algorithm无代码=LOSE, Debug无代码且无bug关键词=LOSE, Complexity无O()=LOSE

  第十九部分：硬件迁移方案(2026-05-20新增)

DGX桌面超算迁移评估:
  代码改动: 几乎不需要改, 只改5个配置常量(5分钟)
  需要改的:
    BitsAndBytesConfig: 4bit NF4 → FP16/BF16
    training_args: batch_size 1→4-8, gradient_accumulation 4→1-2
    MAX_TOKENS: 512→1024-2048
    QUESTIONS_PER_ITER: 1000→2000-5000
    .to("cuda") → .to(model.device) (多卡时需要)
  不需要改的:
    训练逻辑(hermes_alphazero.py) — 纯PyTorch硬件无关
    VSA评分 — 纯NumPy CPU计算
    自博弈框架(Phase 0-4) — 逻辑层不依赖GPU
    soul_memory/MCP/checkpoint/LoRA — 全部硬件无关

  DGX收益(A100 80GB vs V100 16GB):
    FP16训练精度无损(vs 4bit损失5-8%)
    batch=4-8梯度更稳定(vs batch=1)
    5000题/轮同等时间但数据量5倍
    可考虑2灵魂并行(2×40GB)

外挂架构的硬件优势:
  7B+外挂 vs 70B纯权重 → 推理成本差10倍, 效果可能只差10%
  外挂库解决"知道什么"(knowledge), 权重解决"怎么想"(reasoning style)
  16GB够用但有天花板: 4bit是硬伤(训练质量), 不是数据量问题
  32GB买的不是"能不能训"而是"训得好不好"(FP16训练质量提升5-8%)

【目的】
  基于最新JEPA研究(ICML 2026 C-JEPA + V-JEPA 2 + LeWorldModel)，
  在A-JEPA训练并行进行时，提前实现和验证两个关键升级方向。

【核心文件】
  D:\VORTEX_FLAME\five_layer_jepa\causal_jepa_v2.py  主模块(SIGReg+ActionPredictor+ActionCAJEPA)
  D:\VORTEX_FLAME\compare_losses.py                   对比验证实验脚本

【v2 模块清单】

  1. SIGRegLoss (SIGReg)
     - 来源: LeWorldModel (LeCun/Mila/NYU, arXiv:2603.19312)
     - 2-term loss (var+cov), 替代4-term CausalVICReg (sim+var+cov+causal)
     - 2个超参数 vs 4个, 训练更稳定, 单GPU可行
     - 对比实验结论: 总loss低~31倍, grad低~4倍, 速度1.3x快

  2. SIGRegWithPredictionLoss
     - SIGReg + 弱sim_loss(weight=1.0)
     - 折中方案: SIGReg稳定性 + 轻度预测精度信号
     - 适合早期训练(预测器弱), 可逐步降低sim_weight

  3. ActionConditionedCausalPredictor
     - 来源: V-JEPA 2-AC (Meta FAIR, arXiv:2506.09985)
     - 在CausalPredictor输入序列前加action token
     - Action通过2层MLP编码为action_token, cat到slot序列
     - 向后兼容: action=None时等价于原CausalPredictor
     - 参数: 459,136 (action部分仅增加~2,688参)
     - 验证通过: 确定性/action改变预测/向后兼容

  4. ActionCAJEPA
     - 完整的Action-Conditioned Audio JEPA
     - 复用ObjectSlotEncoder + ObjectLevelMasker
     - 内置ActionConditionedCausalPredictor
     - 支持SIGReg或CausalVICReg (use_sigreg=True/False)
     - predict_consequence()方法: 预测动作后果→VORTEX校验
     - 参数: 2,667,264 (含context/target encoder)
     - 验证通过: SIGReg训练/CausalVICReg训练/predict_consequence

【对比实验结论 (compare_losses.py)】
  运行状态: ALL CHECKS PASSED ✅

  Scenario 1 (正常数据):
    CausalVICReg total=~5.7 | SIGReg total=~1.8 | SIGReg+Sim total=~1.9

  Scenario 2 (坍塌数据 std<0.01):
    CausalVICReg var_penalty=~1.97 | SIGReg var_penalty=~0.99
    → SIGReg/CausalVICReg var_loss ratio=0.50x (等比例惩罚)
    → 两者坍塌检测能力相当

  Scenario 3 (梯度比较):
    CausalVICReg grad=~0.53 | SIGReg grad=~0.13 | Ratio=0.25x
    → SIGReg梯度更平滑, 训练更稳定

  Scenario 4 (5步合成训练):

Part 41: 🔧 CAJEPA梯度尖刺根治 + 诊断清理 (2026-06-01)
================================================================================

【根因诊断】
  CAJEPA在Epoch 1-2反复出现梯度尖刺(50-220x)，降低lr到5e-5无效。
  根因: 随机初始化的per_slot_input_proj层造成的分布不匹配。
  per_slot_input_proj是ModuleList[N个Linear]，用于给shared feature上色让Slot Attention竞争。
  随机初始化(Kaiming)的输出方差~10x输入范围，下游Slot Attention/Predictor面临陌生分布→梯度爆炸。

【两个修复】
  (A) 近恒等初始化: W ≈ I + N(0, 0.001), b=0
      移除LayerNorm+GELU(它们会破坏近恒等性质)
      代码: lin.weight.add_(torch.eye(input_dim)) after normal_(std=0.001)
  (B) 分离学习率: main_params=5e-5, slot_proj_params=1e-5
      optimizer = Adam([{"params": other, "lr": base_lr}, {"params": slot_proj, "lr": base_lr/5}])

【训练状态】
  ✅ 梯度spike从50-220降至50-52(仍有，clip到5.0安全范围内)
  ✅ 训练稳定运行: E4 B~830/6663, loss≈2.7, jepa≈2.1, cont≈2.0, lr=4.95e-05, VRAM=0.3GB
  ⏳ 梯度spike仍在~1/100步出现(norm 50+) — 近恒等初始化改善了但未完全消除
    后续可尝试: 增加separated LR差距(1:10)、逐步提高slot_proj LR(从1e-5→2e-5)、
