## 一、设计哲学

### 1.1 为什么不走纯3D

| | Three.js 3D | Canvas 2D + CSS 3D (本方案) |
|---|---|---|
| 首屏加载 | 3-8秒 (140KB+3D初始化) | **< 2.0秒** (D3 CDN ~100KB gzip) |
| 手机性能 | 8帧/秒，发热严重 | **30-60帧/秒** |
| 移动数据流量 | 加载3D纹理 | **0额外纹理流量** |
| 视觉冲击 | 取决于建模精度 | **取决于设计能力** |
| GPU占用 | 持续消耗 | **0 (Phase 1-2完全不碰)** |

### 1.2 Flipbook交互哲学的本质

Flipbook.page真正的革命不在AI生成像素，而在**交互范式**：

```
传统网页:  链接跳转 (A页 → B页 → C页)  ← 碎片化
Flipbook:  无限钻取 (表面 → 深入 → 更深入)  ← 连续感

我们做的是:
  星系全景 → 灵魂特写 → 行业星球 → 技术栈 → AI对话
  每一层都是程序化绘制，不用跳转，像穿越一本无限深的书
```

### 1.3 空间感的来源 (不是3D渲染)

```
空间感 ≠ 3D模型

真正的空间感来自:
  ✅ 层级深度 (Z轴视差，CSS perspective)
  ✅ 运动视差 (不同层不同移动速度)
  ✅ 缩放原点 (从点击处放大，而非居中)
  ✅ 颜色深度 (深层更暗/暖，浅层更亮/冷)
  ✅ 粒子遮蔽 (前景粒子大且快，背景粒子小且慢)
  ✅ 过渡连贯 (没有页面跳转，只有镜头推拉)
  ✅ 阴影投射 (box-shadow / drop-shadow制造悬浮感)
```


## 二、核心交互：无限深度钻取

### 2.1 五层深度模型

```
深度 0: 🌌 全景星云 (所有94节点+163条边散布在2D星场)
   │  ── 点击任意灵魂恒星 ──→
   ▼
深度 1: 🌟 灵魂星系 (该灵魂+绑定行业+关联基础学科，集中高亮)
   │  ── 点击任意行业行星 ──→
   ▼
深度 2: 🪐 行业星球 (key_techs卡片列表, 技术栈展开)
   │  ── 点击某个技术卡片 ──→
   ▼
深度 3: 🔬 技术详情 (key_techs展开占位: 描述+相关学科+应用场景)
   │  ── 点击"咨询专家" ──→
   ▼
深度 4: 💬 AI对话面 (触发对应灵魂模块, Phase 3实现)

← 每层可回退 (ESC/返回按钮/双指捏合)
```

**深度3数据策略**: 
- 现有JSON的L4节点有 `key_techs` 数组（如`["光刻工艺","刻蚀工艺","薄膜沉积","离子注入","封装测试","EDA工具"]`）
- v1.0 直接展示这些技术项作为可点击卡片
- 技术详情用程序化模板填充（技术名+所属行业+关联灵魂+简短描述），不依赖额外数据

### 2.2 钻取过渡动画

```
Canvas层过渡 (JS requestAnimationFrame插值):
  0ms-200ms:  起点节点放大+发光脉冲
  200ms-600ms: 非目标节点淡出(alpha→0), 粒子加速飞过(速度感)
  400ms-900ms: 目标内容从深处浮现(scale 0.3→1.0)
  600ms-1000ms: 完成过渡

DOM层过渡 (CSS transition):
  0ms-300ms:   当前面板 fadeOut
  400ms-700ms: 新面板 slideIn + fadeIn
```

### 2.3 鼠标/触摸尾迹粒子

```
Canvas上始终跟随指针的粒子尾迹:
  - 15-20个粒子组成尾巴
  - 每个粒子: 2-4px发光圆点
  - 颜色随当前选中灵魂的Tier主题色
  - 渐隐生命周期: 0.8秒
  - 手机端: 触摸点同理
```

### 3.5 动画循环 (requestAnimationFrame)

```
每帧执行:
  1. 清空canvas
  2. D3力模拟 tick (持续收敛)
  3. 更新+绘制背景粒子
  4. 绘制202条边线 (按类型分色)
  5. 绘制32条soul_bind边的流动粒子 (更新位置)
  6. 绘制101个节点 (带pulse动画)
  7. 更新DOM标签位置 (跟随节点坐标)
  8. 绘制鼠标尾迹
  9. 如果正在钻取过渡: 摄像机矩阵变换

降级策略 (帧时间>33ms时):
  第1级: 关闭流动粒子
  第2级: 背景粒子减半
  第3级: 关闭粒子闪烁
```

## 四、技术架构

### 4.1 依赖策略

```
单HTML文件: index_v6.html

外部依赖 (1个CDN):
  ✅ D3.js v7 (CDN) ~100KB gzip
    用途: 力模拟布局 (94节点+163边的复杂关系网)
    理由: 纯环形布局丢失L1↔L2↔L3↔L4横向连接,
          D3力模拟可完整展示5种边线关系
    备选: 可在Phase 1先用D3, Phase 2优化为固定坐标JSON

零构建工具:
  - 无npm/webpack/vite
  - 浏览器直接打开即可运行
  - 数据内嵌<script>标签 (与现有HTML模式一致)

体积估算:
  - D3.js (CDN): ~100KB gzip (浏览器缓存, 二次加载0)
  - HTML + CSS + JS: ~25KB
  - 内嵌数据: ~15KB
  - 总计首次: ~140KB, 二次: ~40KB
```

- 线上系统永远只接触"加密封装接口"，永远接触不到核心明文

### 11.2 第二层：全域零信任+内网闭环+外网零暴露

核心原则：根本不给你入口

- 门店/后台全内网闭环，外网零直连零端口暴露
- 外网访问走加密堡垒机+动态跳板+一次性令牌
- 四重校验：身份→设备→行为→密钥
- 全链路国密SM2/SM4+AES-256双加密，会话密钥毫秒级轮换

### 11.3 第三层：算力硬件绑定+内存运行+动态熔断

核心原则：脱离你的硬件，寸步难行

- CPU+GPU+主板+自定义硬件密钥，不可篡改指纹
- 核心模型硬盘全加密，只在内存中解密运行，关机/断电即销毁
- 内核级熔断：检测异常→瞬间清空内存+切断内网+销毁密钥+全网报警

### 11.4 第四层：开发者生态沙箱锁死+利益绑定+行为风控

核心原则：技术锁死+利益绑定+行为监控三重兜底

- 开发/调试/打包全在封闭沙箱，不可导出
- 应用强制嵌入硬件指纹校验+平台签名+算力心跳，脱离平台闪退
- 收益深度绑定：分成+流量+算力+专利+合规
- 后台实时监控异常代码导出/逆向调试/批量复制/陌生硬件接入

### 11.5 第五层：主动反制体系

核心原则：不是被动防，是主动预警+溯源+反制

- 全域蜜罐：高仿真节点，黑客试探即触发预警，记录攻击源/手法/路径
- 攻击溯源：结合网安部门/公安/国际情报，跨国APT溯源取证上报
- 动态防御：检测到APT→紧急密钥轮换+硬件规则升级+熔断可疑节点+调整内网拓扑

### 11.6 第六层：法律+主权+专利兜底

核心原则：国家级黑客也不敢明目张胆

- 发明专利+软著+商业秘密三重保护
- 境外APT涉嫌违反《网络安全法》《数据安全法》
- 跨国攻击→上报网安+公安+关键基础设施保护机构，启动国家级溯源反制

### 11.7 终极结论

- **技术层面**：核心物理隔绝+网络零入口+算力硬件绑定+行为熔断+主动反制，国家级APT只能碰外围
- **商业层面**：线下门店+算力网络+开发者生态+专利壁垒+用户信任，黑客永远无法复制
- **主权层面**：中国境内运营，法律+主权+网安体系兜底，境外攻击有明确追责路径
