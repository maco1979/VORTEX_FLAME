# Cezanne 7B 重训设计文档

## Step 1: 需求澄清

1. **要解决什么问题？** Base 77% > 所有LoRA版本（S1 52%, S2 67%, S3 65%, S4 69%），LoRA r=16灾难性遗忘
2. **为什么现在要做？** 不做则7B Cezanne永远比Base差，无法涌现
3. **预期结果？** S2 ≥ 75%（接近Base），S3 ≥ 80%（超越Base）
4. **影响组件？** 训练数据(soul_training_data/cezanne/) + LoRA权重(soul_lora_v2/cezanne/) + 训练脚本(pipeline_8b/)
5. **最坏情况？** r=8仍然遗忘 → 回滚到Base，尝试r=4

## 重训方案

### 核心改变：r=16 → r=8

旧方案 r=16 改动太多参数 → 灾难性遗忘
新方案 r=8 只改一半参数 → 保留Base能力

### S2: 逻辑学 + S1不足补充（从Base直接训）

**目标：补齐debug/logic2/formal/hoare四大弱点，同时保持Base的sort/graph能力**

训练数据设计（~50条高质量）：

   - 并发race condition (2条)
   - 内存泄漏 (2条)

2. **形式逻辑** (10条) — 针对logic2≤25%, formal≤40%
   - 假言三段论 P→Q, Q→R ⊢ P→R (2条)
   - 反证法 (2条)
   - 逆否命题 (2条)

3. **图算法** (8条) — 针对graph退化

4. **排序算法** (6条) — 针对sort退化
   - 快排Lomuto/Hoare分区 (2条)
   - 归并排序 (2条)
   - 堆排序 (2条)

5. **编程核心** (14条) — 保持Base能力
   - 树操作 (2条)
